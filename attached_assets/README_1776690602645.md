## Your Role vs. Claude's Role

**Claude has written:** All application code — components, screens, API routes, data models, types, business logic, and configuration.

**Your role (Replit):** Infrastructure only.
- Spin up the Node.js backend server
- Provision and connect the PostgreSQL database
- Provision and connect Redis
- Set environment variables
- Configure deployment pipeline
- Run database migrations
- Wire the frontend to the backend API

**Do not rewrite, regenerate, or modify application code.** The code in this package is intentional and complete. If something doesn't work, raise it — don't replace it.

## Database Setup

Prisma handles migrations. Run in order:

```bash
# Install dependencies
npm install

# Generate Prisma client
npx prisma generate

# Run migrations (creates all tables)
npx prisma migrate deploy

# Optional: seed with sample curated plans
npx prisma db seed
```

**Never run `prisma migrate reset` in production.** It drops all data.

## Third-Party Service Setup

### 1. Anthropic (AI — Required Day 1)
- Create account at console.anthropic.com
- Generate API key
- Set `ANTHROPIC_API_KEY` in `.env`
- Model: always use `claude-opus-4-6` for best quality
- Budget: expect ~$0.01–0.05 per Kitchen Wizard run

### 2. Stripe (Payments — Required Before Paid Tier Launch)
- Create account at stripe.com
- Create two products in Stripe dashboard:
  - "Kiwi Premium Monthly" — $9.99/month recurring
  - "Kiwi Premium Annual" — $100/year recurring
- Copy Price IDs to `.env`
- Set up webhook endpoint: `POST /webhooks/stripe`
- Required webhook events: `customer.subscription.created`, `customer.subscription.updated`, `customer.subscription.deleted`, `invoice.payment_succeeded`, `invoice.payment_failed`
- Install Stripe CLI locally for webhook testing: `stripe listen --forward-to localhost:3001/webhooks/stripe`

### 3. Twilio (SMS OTP — Required for Phone Signup)
- Create account at twilio.com
- Buy a phone number (~$1/month)
- Set `TWILIO_ACCOUNT_SID`, `TWILIO_AUTH_TOKEN`, `TWILIO_PHONE_NUMBER`
- SMS cost: ~$0.0079 per message

### 4. Resend (Email — Required Day 1)
- Create account at resend.com
- Verify your sending domain (getkiwi.app or equivalent)
- Generate API key
- Free tier: 3,000 emails/month — sufficient for early beta

### 5. Instacart (Grocery API — Apply for Access)
- Apply at instacart.com/business/partnerships
- This requires approval — apply early as lead time is 2–4 weeks
- In the meantime, the RPA adapter for Whole Foods will be the primary grocery integration
- Set `INSTACART_API_KEY` once approved

### 6. Expo (Mobile Builds + Push Notifications)
- Create account at expo.dev
- Run `npx eas build --platform all` for first build
- Push notifications use Expo Push service — no additional setup required
- Apple Developer Program ($99/year) required for TestFlight and App Store
- Google Play Console ($25 one-time) required for Play Store

## AI Integration Notes

All AI calls go through `/server/src/ai/`. **Never call the AI API directly from routes or services.** Always use the orchestration layer.

The orchestration layer:
1. Receives structured input from domain services
2. Builds a safe, constrained prompt
3. Calls the Claude API
4. Validates output against a JSON schema
5. Returns structured records — never raw text

If schema validation fails: retry once with stricter formatting instruction, then return a safe error state. **Never let malformed AI output reach the UI.**

Example call (from plan service):
```typescript
import { wizardOrchestrator } from '../ai/wizard';

const candidates = await wizardOrchestrator.buildPlans({
  householdSize: 4,
  wantsLeftovers: true,
  difficulty: 'medium',
  weeklyPacing: 'mixed',
  dietaryRestrictions: ['high_protein'],
  cuisinePreferences: ['american', 'mexican'],
  prompt: 'easy family dinners with one nicer meal Friday'
});
// Returns: WizardPlanCandidate[] — validated, structured, UI-ready
```

## Grocery Retailer Integration Notes

The retailer adapter pattern means the UI never changes regardless of which backend integration is used.

```typescript
// All retailers implement this interface:
interface RetailerAdapter {
  authenticate(userId: string, credentials: any): Promise<AuthResult>;
  searchItems(query: string): Promise<RetailerItem[]>;
  addToCart(items: NormalizedGroceryItem[]): Promise<CartResult>;
  getCartUrl(): string;
}
```

**Instacart:** Use the official API once approved. See `instacart.adapter.ts`.

**Whole Foods / Amazon Fresh:** Uses Playwright browser automation. This adapter runs in a background job queue (Redis + BullMQ), not in the request-response cycle. The user gets a "processing" state and a webhook fires when complete. See `wholefood.adapter.ts`. Keep this adapter isolated — never mix browser automation code into the core service layer.

**Peapod / Stop & Shop:** Investigate their current integration options first. Ahold Delhaize acquired Peapod — their tech stack may have an affiliate/cart URL pattern. Check before defaulting to RPA.

## Subscription & Entitlement Logic

Feature gating is handled by the `SubscriptionService`, not by checking raw billing data in routes.

```typescript
// Always check entitlements this way:
const canOrder = await subscriptionService.can(userId, 'grocery_ordering');
const canUnlimitedPlans = await subscriptionService.can(userId, 'unlimited_plans');

// Never do this:
const user = await db.user.findUnique(...);
if (user.subscriptionStatus === 'active') { ... } // ❌ wrong
```

**Free tier limits:**
- Max 4 saved plans (track `deleted_plan_count` for conversion signals)
- No online grocery ordering
- Basic cooking steps only (no full prep flow)
- Ads shown

**Premium:**
- Unlimited plans
- Grocery ordering
- Full cooking intelligence
- Ad-free

**Trial:** 30 days full Premium access. Duration is controlled by `DEFAULT_TRIAL_DAYS` env var — set to 0 to disable trials for new signups.
